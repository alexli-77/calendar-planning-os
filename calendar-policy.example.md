## Energy Windows

<!-- When do you usually do your best work? -->

- Deep work: 09:30-11:30
- Admin / follow-up: 14:00-15:00
- Review / planning: 20:00-20:30

## Scheduling Style

<!-- How full should a day feel? -->

- Prefer 2-3 meaningful work blocks per day.
- Leave at least 30 minutes between demanding blocks.
- Do not fill the whole day.
- Batch small admin tasks together.
- Weekly planning should create a rough shape first.
- Daily planning can move blocks based on what actually happened.

## Protected Time

<!-- Times the planner should avoid unless explicitly told otherwise. -->

- Keep lunch open.
- Keep late evening light.
- Do not schedule deep work after 21:00.

## Task Rules

<!-- How should tasks become calendar blocks? -->

- P0/P1 tasks get the first deep work windows.
- Email, reimbursements, and planning updates go into admin blocks.
- Weekly review should stay as a review block, not deep work.
- If a task has a due date, try to schedule it before or on that date.
- If a task has a preferred date, try that date first.
- If the day already feels full, leave the task as a warning instead of overpacking.

## Optional Advisory Skills

<!-- Skills advise; the calendar decision policy makes the final call. -->

- Use `energy-time-control` for energy windows, recovery, and block shape.
- Use `cognitive-clarity-planning` before scheduling vague work.

## Calendar Writeback

<!-- Writeback is disabled in the current alpha. Keep this as a future preference. -->

- Draft first.
- Ask before writing.
- Never overwrite existing events automatically.
- Show conflicts before confirmation.
