# Cognitive Clarity Planning

Use this skill as an advisory layer. It decides whether a task is ready to become calendar time.

## Workflow

1. Inspect each task title, source, goal, due date, and expected output.
2. Mark tasks as `ready`, `needs_clarification`, or `needs_breakdown`.
3. Prefer scheduling concrete next actions over broad project names.
4. Add a clarification block when the task is important but vague.
5. Add review or feedback checkpoints when a task depends on another person or external validation.
6. Return recommendations and warnings to the calendar decision layer.

## Ready To Schedule

A task is ready when it has:

- a concrete next action,
- an expected output,
- a rough duration,
- enough context to start,
- a reason it matters now.

## Not Ready

Do not schedule a large deep-work block for vague items such as:

- "work on project",
- "think about plan",
- "handle portfolio",
- "need to confirm this",
- "research topic" with no question or output.

Schedule a short clarification block instead.
