# Cognitive Clarity Planning Rules

These rules are distilled from cognitive planning and action notes. Keep them advisory unless the user's calendar decision policy makes them mandatory.

## Clarity Gate

Before scheduling a task, ask:

- What is the next physical or digital action?
- What output should exist after the block?
- How long is the first useful block?
- What information is missing?
- Who or what provides feedback?

If these questions cannot be answered, schedule clarification before execution.

## Task States

- `ready`: clear action, output, duration, and timing.
- `needs_clarification`: important but vague; schedule 15-30 minutes to define the next action.
- `needs_breakdown`: too large; split into smaller blocks.
- `needs_feedback`: depends on review, response, or external validation; schedule follow-up or review.

## Calendar Recommendations

- Replace vague project names with concrete deliverables.
- Prefer "draft one-page review sheet" over "work on interview prep".
- Prefer "list five open questions" over "think about application".
- Prefer "send version for review" over "portfolio".
- Put feedback loops on the calendar when the task depends on another person.

## Planning Principles

- The purpose of a plan is to make the next action visible, not to control the whole day.
- Fewer clear blocks are better than many vague blocks.
- If a task feels hard because it is fuzzy, reduce it to a short clarification block.
- When learning or creative work matters, schedule output and feedback, not only input.
- Use review blocks to convert what happened into the next calendar decision.

## Output To Calendar Decision Layer

Return:

- readiness state,
- rewritten concrete task title when useful,
- recommended block type,
- suggested duration,
- missing information,
- feedback or review checkpoint if needed.
